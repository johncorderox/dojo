Javascript Basics

Javascript is a cross-platform, object-oriented scripting language. Javascript is extremely popular for a variety of reasons. It is a small and lightweight languge allowing maximum flexibilty for developers to take it in a bunch of different directions. Javascript lives inside a shost enviorment (a web browser or Node Server), it can be connected to the objects of these enviorments to provide programmatic control over them.

  • Variable Declaration

    • Javascript variables are containers for storing data values- imagine a cup you fill with coffee, the cup holds the coffee, a variable holds a value .All Javascript variables must be indentified with unique names. These unique names are called identifies.

    var x;

  • Variable Assignment

    • Assignment operators assign values to Javascript variables - our cup can now have coffee poured in it, giving our variable a value to hold. The = assignment operator assigns a value to a variable.

    var x = 10;

  • Data Types

    • Data Types are an important concept; to be able to operate on variables you need to know the data type. There are six data types that are Javascript primitives: Boolean - true or false; null - null aka nothing; Number - 42 or 3.13159; String - "Coding Dojo Rocks!"; Array - [1, 'Coding', 2, 'Dojo']; and Object - {first_name: 'Jane', last_name 'Doe'}



  • Conditionals

    • When you write code, you want to preform different actions for different decisions - hitting different code blocks based on values or conditions that have been met. You can use conditonal statements in your code to accomplish this. There are the following conditional statements: if a specified condition is true, do this code in our code block; else if to specifiy a new condition to test, if the first condition is false; else we execute this block of code;



  • Loops

    • There are many different kinds of loops in every programming language, but they all essentally do the same thing; they will repeat an action some number of times. Imagine you have to run a mile, well you run around the track four times and then you stop. Thats a loop!



  • Functions

    • Functions are an ecapsulation of a code block. When we call our function this will run that code block. Think of it as a list of instructions. As an example imagine we are putting toegther a desk from Ikea, we open up the insutrctions manual and we get started, first we screw the legs on the table top; next we place the table the right way up. Done! We finished our instructions. Sadly there are a ton more tables to do so lets our function over and over and over again.
